# LaTeX draft for `Zero-Knowledge Verifiable Offline DQN Training from Committed Trajectories`

This folder contains the current LaTeX draft of the paper, synchronized with the repository's present implementation state.

## Current status

The project is no longer only at the proposal stage. The repository now includes:

- an offline RL experimental pipeline on CartPole-v1,
- committed-data artifacts based on Merkle-style membership proofs,
- a pre-ZK artifact-verifier prototype for:
  - committed transition membership,
  - Double-DQN-style Bellman target correctness,
  - SmoothL1 TD loss correctness,
  - minibatch-average TD loss correctness,
  - checkpoint anchoring via SHA-256.

This is **not yet a full zero-knowledge proof-of-training system**. It should currently be described as a **pre-ZK artifact-verification prototype** that formalizes the MVP statement needed for a future proving backend.

## Files

- `main.tex`: main manuscript entry point
- `sections/`: section files
- `refs.bib`: bibliography file
- `figures/`: figures used by the draft

## What the paper should currently claim

At the present stage, the paper should claim that the repository provides:

- an empirical offline RL study showing that dataset structure strongly affects offline value learning,
- a concrete verification-oriented formulation based on committed offline data,
- a prototype verifier for membership + TD-target/loss correctness,
- a clear roadmap from verified TD arithmetic to one verified DQN update step.

The paper should **not** yet claim:

- a complete zero-knowledge proving backend,
- a full proof of training,
- certified optimizer updates across full training,
- a proof that the published checkpoint was generated by a fully verified training trace.

## Recommended next technical milestone

The next research milestone should be:

> one verified offline DQN update step

That step should extend the current MVP from:
- committed minibatch membership,
- Bellman target correctness,
- SmoothL1 TD loss correctness,

to also include:
- gradient computation,
- optimizer update,
- pre-update and post-update checkpoint anchoring.

## Suggested citation / framing note

Until a real proving backend is added, phrases such as:

- “proof of training”
- “zero-knowledge proof of offline DQN training”

should be used carefully and qualified in the text.

Preferred phrasing at this stage is:

> pre-ZK artifact-verifier prototype for committed-data membership and TD-arithmetic correctness.

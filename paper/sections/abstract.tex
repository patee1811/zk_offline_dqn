Offline reinforcement learning creates a verification problem that is different
from supervised proof-of-training: a dishonest prover can tamper not only with
model parameters or losses, but also with replay membership, terminal flags,
Bellman targets, action selection, target-network values, and minibatch
aggregation. We present proof-oriented verification of selected offline-DQN
artifacts over committed replay data. The system is intentionally scoped: it
does not claim a full proof of training from initialization to final checkpoint.
Instead, it decomposes the DQN training stack into auditable relations for
committed transition membership, fixed-point Bellman/TD computation, distinct
minibatch aggregation, model-grounded MLP forward/argmax evidence, and a tiny
one-step SGD update extension.

The implemented artifact pipeline commits replay transitions with SHA-256
Merkle roots, exports canonical fixed-point witnesses, checks adversarially
tampered fixtures with a Python semantic oracle, and validates one SP1 zkVM
proof path for the canonical TD MVP vector. The generated report aggregate
contains 29 benchmark rows and 21 tamper rows, with provenance in
\path{artifacts/reports/final_ndss/paper_numbers.json}. The supported SP1
proof claim is deliberately narrow: on Kaggle, the TD MVP SP1 backend proves
and verifies \path{zk_backend/test_vectors/td_mvp_case_0.json}, with
167.726006 s guest proving time, 0.190326 s verification time, 2,783,869 proof
bytes, and 385,048 cycles.

The contribution is therefore a reproducible security/systems artifact rather
than an ML performance claim. It formalizes which public claims and private
witnesses are checked at each layer, identifies the exact backend used by each
relation, and packages reviewer-facing reproduction commands for fast Python
semantic-oracle checks and opt-in TD MVP SP1 validation on Linux, WSL2 Ubuntu,
or Kaggle.

\section{Reproduction Commands}
\label{app:reproducibility}

Run the fast Python smoke path from the repository root:
{\scriptsize
\begin{verbatim}
python scripts/experiments/run_full_regression.py
python scripts/experiments/check_report_sources.py
python scripts/experiments/generate_paper_reports.py
\end{verbatim}
}

Run the scoped TD MVP SP1 validation on Linux, WSL2 Ubuntu, or Kaggle with SP1
installed:
{\tiny
\begin{verbatim}
cd zk_backend/td_mvp/sp1
cargo test
cargo run --release -p td-mvp-host -- --execute
RUN_SP1_PROVE=1 cargo run --release -p td-mvp-host -- --prove
\end{verbatim}
}

The Kaggle wrapper can request the same proof path through:
{\tiny
\begin{verbatim}
python scripts/experiments/run_phase6_kaggle_validation.py --use-local-archive --run-sp1-setup --run-sp1-execute --run-sp1-prove
\end{verbatim}
}

\section{Tamper-Test Definitions}
\label{app:tamper-tests}

The final tamper matrix includes distinct-batch attacks on duplicate index,
wrong item index, swapped item order, wrong item loss, wrong claimed batch
average, and wrong path order. It includes forward-TD attacks on online model
weight, target model weight, activation, ReLU mask, argmax, selected target
value, and claimed batch loss. It includes tiny-update attacks on gradient
tensor, delta tensor, learning rate, post model weight, post model commitment,
and SmoothL1 gradient. The broader regression suite also retains legacy TD MVP
tamper tests for schema, reward, done, rounding, Merkle, target, and TD-error
cases.

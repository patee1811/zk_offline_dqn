We presented a concrete research path toward verifiable offline DQN training from committed data. Instead of attempting to solve general RL verifiability at once, we first established an offline benchmark pipeline on CartPole-v1, studied how different datasets interact with different learning paradigms, and then implemented a pre-ZK artifact-verifier prototype for the core training computations of interest.

Our empirical findings are clear. A narrow high-return behavior dataset is excellent for behavior cloning, but naive offline Double DQN collapses on it. A conservative Q-learning objective~\cite{kumar2020cql} substantially recovers performance, making it a stronger candidate for future verifiable training than naive offline Q-learning on narrow behavior data. Conversely, broader-coverage data is much more favorable to offline bootstrapped value learning, highlighting the importance of dataset structure for any meaningful verifiability claim.

These observations matter directly for the verification agenda. Verifiability is only useful if the certified computation is itself meaningful, stable, and tied to a fixed committed dataset. Accordingly, the present repository now supports verification of committed-sample membership, Double-DQN-style Bellman target correctness, SmoothL1 TD loss correctness, minibatch-average loss correctness, public checkpoint anchoring through SHA-256, and one-step SGD update consistency including gradient recomputation and parameter-delta verification.

\paragraph{Future work.}
This paper represents an ongoing effort, and several important extensions remain. The immediate next step is to move from one verified update step to \emph{multi-step verified training traces}, chaining pre/post checkpoint hashes across consecutive SGD steps and explicitly handling target-network synchronization events within the proof statement structure.

Beyond multi-step traces, future work includes:
\begin{itemize}[leftmargin=1.5em]
    \item \textbf{ZK backend integration:} translating the current fixed-point arithmetic and statement structure into an actual proving backend such as a zkSNARK system~\cite{groth2016zksnark, gabizon2019plonk};
    \item \textbf{Larger environments:} extending the pipeline beyond CartPole-v1 to environments with higher-dimensional observations and larger action spaces;
    \item \textbf{Optimizer generalization:} supporting Adam and other adaptive optimizers within the verification statement;
    \item \textbf{Replay-sampling verification:} proving that minibatch indices were chosen according to a specified sampling distribution;
    \item \textbf{Proof composition:} using recursive proof techniques to compose many single-step proofs into a complete proof of training.
\end{itemize}

The next milestone is therefore not ``general RL + ZK,'' but a verified multi-step offline DQN training trace with target-network synchronization. That stronger goal will require formalizing public and private inputs for sequences of checkpoints, representing gradients and optimizer arithmetic in a proof-friendly form, and extending the current prototype into a genuine proving backend.

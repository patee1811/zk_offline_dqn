\subsection{Artifact and Environment}

All experiments use the repository artifacts locked in the final Phase E
package. The
canonical TD test vector is
\path{zk_backend/test_vectors/td_mvp_case_0.json}. The Python semantic
oracle is \path{scripts/artifacts_export/verify_td_mvp_test_vector.py}. The
SP1 backend is the Rust workspace in \path{zk_backend/td_mvp/sp1/}.

The concrete environments are CartPole-v1 and MountainCar-v0~\cite{brockman2016gym}.
The SP1 backend proves distinct TD, forward-TD MLP, and tiny one-step SGD
relations at the scales reported in the final benchmark matrix.

\subsection{Reproducibility Commands}

The Python regression suite is run with:
{\scriptsize
\begin{verbatim}
python scripts/experiments/run_full_regression.py
\end{verbatim}
}
It performs compile checks, TD/minibatch verification, forward TD consistency,
one-step verification, short-trace verification, and negative tests.

The SP1 benchmark is run on Linux, WSL2 Ubuntu, macOS, or Kaggle with:
{\scriptsize
\begin{verbatim}
python3 scripts/experiments/benchmark_distinct_td_sp1.py --prove
python3 scripts/experiments/benchmark_forward_td_mlp_sp1.py --prove
python3 scripts/experiments/benchmark_mountaincar_forward_td_sp1.py --prove
python3 scripts/experiments/benchmark_one_step_sgd_tiny_sp1.py --prove
python3 scripts/experiments/run_final_ndss_regression.py
\end{verbatim}
}
The runners generate accepted fixtures, prove the selected accepted cases,
execute tampered cases, compare Python and SP1 outcomes, and write JSON, CSV,
and Markdown summaries.

The SP1 negative suite can also be run directly from the SP1 workspace:
{\scriptsize
\begin{verbatim}
cd zk_backend/td_mvp/sp1
bash run_negative_cases.sh
\end{verbatim}
}

\subsection{Benchmark Cases}

The accepted benchmark matrix contains:
\begin{itemize}[leftmargin=1.5em]
    \item TD-1: one Merkle membership plus one TD/SmoothL1 check;
    \item TD-2, TD-4, TD-8: distinct replay minibatches with multiple Merkle
    memberships, public ordered leaf indices, per-item TD/SmoothL1 checks, and
    integer batch-average loss;
    \item CartPole and MountainCar forward-TD MLP batch-1 proofs;
    \item CartPole tiny one-step SGD batch-1 proof.
\end{itemize}

The adversarial suite includes single-transition tampering of schema version,
reward, fixed-point rounding, done flag, done-branch semantics, observation,
leaf encoding, Merkle path, leaf index, path order, target-network value,
claimed target, claimed loss, leaf hash, and TD error. Batch tampering covers
claimed batch loss, batch size, item loss, item index, duplicate index, swapped
item order against public leaf indices, path order, target-network value, and
fixed-point rounding.

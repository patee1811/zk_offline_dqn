\subsection{Artifact and Environment}

All experiments use the repository artifacts locked at the end of Week 5. The
canonical TD test vector is
\path{zk_backend/test_vectors/td_mvp_case_0.json}. The Python semantic
oracle is \path{scripts/artifacts_export/verify_td_mvp_test_vector.py}. The
SP1 backend is the Rust workspace in \path{zk_backend/td_mvp/sp1/}.

The concrete environment is CartPole-v1~\cite{brockman2016gym}. The model used
by the Python extension artifacts is a multilayer perceptron with input
dimension 4, two hidden layers of width 128, and two output actions. The SP1 TD
backend does not prove this neural-network forward pass; it proves consistency
of the fixed-point TD witness values and committed transition membership.

\subsection{Reproducibility Commands}

The Python regression suite is run with:
{\scriptsize
\begin{verbatim}
python scripts/experiments/run_full_regression.py
\end{verbatim}
}
It performs compile checks, TD/minibatch verification, forward TD consistency,
one-step verification, short-trace verification, and negative tests.

The SP1 benchmark is run on Linux, WSL2 Ubuntu, macOS, or Kaggle with:
{\scriptsize
\begin{verbatim}
python3 scripts/experiments/benchmark_sp1_td_mvp.py \
  --prove
\end{verbatim}
}
The runner generates TD-1/2/4/8 accepted fixtures, generates proofs for the
accepted cases, executes tampered cases, compares Python and SP1 outcomes, and
writes JSON, CSV, and Markdown summaries.

The SP1 negative suite can also be run directly from the SP1 workspace:
{\scriptsize
\begin{verbatim}
cd zk_backend/td_mvp/sp1
bash run_negative_cases.sh
\end{verbatim}
}

\subsection{Benchmark Cases}

The accepted benchmark matrix contains:
\begin{itemize}[leftmargin=1.5em]
    \item TD-1: one Merkle membership plus one TD/SmoothL1 check;
    \item TD-2, TD-4, TD-8: multiple Merkle memberships, per-item TD/SmoothL1
    checks, and integer batch-average loss.
\end{itemize}

The adversarial suite includes single-transition tampering of schema version,
reward, fixed-point rounding, done flag, done-branch semantics, observation,
leaf encoding, Merkle path, leaf index, path order, target-network value,
claimed target, claimed loss, leaf hash, and TD error. Batch tampering covers
claimed batch loss, batch size, item loss, item index, path order,
target-network value, and fixed-point rounding.

\subsection{Pre-ZK Extension Benchmarks}

For context, the repository also retains Python benchmarks for one-step update
verification and 8-step short-trace verification. These are not SP1 proof
benchmarks. They are reported to clarify the engineering path from TD proofs
toward proof-of-update and proof-of-trace.

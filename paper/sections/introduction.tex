Offline reinforcement learning (offline RL) trains policies from a fixed replay
dataset without additional environment interaction~\cite{levine2020offline}.
This fixed-data setting is attractive when interaction is costly, unsafe, or
private, but it also exposes a verification gap. A model producer can claim
that an offline DQN audit or update used a particular replay dataset while
silently changing a transition, terminal flag, Bellman target, target-network
value, loss, or minibatch aggregation. A verifier who only sees a final
checkpoint has little evidence that the reinforcement-learning computation was
performed over the committed data.

Zero-knowledge proof systems and zkVMs offer a way to make such claims
checkable without revealing all private replay contents or intermediate model
values. Existing proof-of-learning and proof-of-training work mainly targets
supervised or general DNN training~\cite{jia2021proofoflearning,zkdl2023,kaizen2024,zkpotvicinity2025}.
Offline DQN has additional statement structure: replay membership, terminal and
non-terminal Bellman branches, Double-DQN action selection, target-network
evaluation, SmoothL1 TD losses, and replay minibatches. These are not merely
engineering details; they define what it means for the computation to be a DQN
training computation.

This paper studies a scoped, artifact-driven target: layered ZK-verifiable
evidence for offline DQN over committed replay data. The current repository
implements Python semantic oracles for distinct minibatch TD, model-grounded
forward-TD MLP checks, and a tiny one-step SGD update. The validated SP1 proof
claim is narrower: proof generation and verification passed on Kaggle for the
TD MVP SP1 backend using \path{zk_backend/test_vectors/td_mvp_case_0.json}.
This separation of relation layers is deliberate. It lets the paper make
precise statements about what has been validated today while avoiding a claim
that the entire training trace has been proved.

The decomposition point is the Bellman/TD layer. It is small enough to prove in
a general-purpose zkVM, yet it captures RL-specific semantics absent from
ordinary supervised losses. Once committed transition membership and TD
arithmetic are anchored, later layers can add neural forward passes, argmax
semantics, gradients, optimizer updates, and recursive trace composition.

The paper makes four contributions:
\begin{itemize}[leftmargin=1.5em]
    \item It defines a threat model and formal relation table for committed
    replay membership, distinct minibatch TD, forward-TD MLP anchoring, and
    tiny one-step SGD update evidence.
    \item It implements a reproducible artifact pipeline with canonical
    fixed-point serialization, SHA-256 Merkle commitments, Python semantic
    oracles, SP1 host/guest/shared crates, and adversarial tamper fixtures.
    \item It reports a benchmark package whose numbers are sourced from
    \path{artifacts/benchmarks/final_ndss/summary.json},
    \path{benchmark_matrix.csv}, and \path{tamper_matrix.csv}.
    \item It gives reviewer-facing reproduction paths: a fast Python smoke path
    and opt-in TD MVP SP1 validation commands for Linux, WSL2 Ubuntu, or
    Kaggle.
\end{itemize}

The claim is intentionally narrower than full proof-of-training. We do not
prove data-collection honesty before commitment, Adam updates, long recursive
training traces, model selection, or policy performance. The contribution is a
security/systems artifact for verifiable RL computation layers, with explicit
boundaries between the TD MVP SP1 proof, Python smoke checks, and future
training-trace composition work.

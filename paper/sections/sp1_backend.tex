The backend lives under \path{zk_backend/td_mvp/sp1/} and follows the standard
host/guest/shared split. The shared crate defines typed public inputs, private
witnesses, Merkle steps, fixed-point helpers, SHA-256 hashing, and TD
arithmetic for the backend-validated TD MVP path. The guest reads a typed input
and calls the shared verifier. The host loads JSON fixtures, selects execution
or proof mode, optionally disables host prechecks for tamper cases, verifies
proofs for accepted cases, and records metrics.

The validated SP1 proof claim is scoped to the canonical TD MVP vector
\path{zk_backend/test_vectors/td_mvp_case_0.json}. For that vector, the guest
recomputes the leaf, leaf hash, Merkle root, terminal branch, target, TD error,
and SmoothL1 loss, and the host records proof generation and verification
status.

The Python artifact layer also contains distinct minibatch TD, forward-TD MLP,
and tiny one-step SGD relations. Those layers are checked by deterministic
Python oracles and represented in report matrices, but they are not claimed
here as validated SP1 proofs unless a separate backend validation summary
records proof generation and verification for the corresponding fixture.

The current proof is intentionally micro-scale and relation-level. It does not
cover a long training trace, target-network synchronization, Adam state,
sampling over a full run, or model selection.

The design has three layers. The first layer is a canonical artifact pipeline:
datasets are serialized into fixed-point transition leaves, committed with a
Merkle tree, and exported as JSON fixtures with public and private fields
separated. This layer is shared by all relations.

The second layer is the semantic oracle. Python exporters and verifiers build
deterministic fixtures for accepted cases and adversarially modified cases.
This oracle is used to validate fixed-point rounding, membership paths, loss
recomputation, model commitments, forward witnesses, and update witnesses. It is
also the fast reviewer path for machines without a prover.

The third layer is the SP1 backend. For SP1-supported relations, the same JSON
fixtures are loaded by a Rust host, passed to a guest program, checked inside
the zkVM, and optionally proved. Negative cases skip host prechecks so that
rejection is attributable to guest relation logic rather than to JSON filtering.

The final artifact package is an aggregation layer over these component
benchmarks. It writes
\path{artifacts/benchmarks/final_ndss/summary.json},
\path{benchmark_matrix.csv}, \path{tamper_matrix.csv}, \path{summary.md}, and
\path{reproduction.md}. The paper uses these files as the source of benchmark
numbers.
